# Dataset Collection

## 개요

본 문서는 AI 그림책 보조 서비스를 위한 객체 탐지 데이터셋 수집 기준을 정의한다.

서비스는 사용자가 손가락으로 가리키는 동화책의 그림 객체, 촉각 교구, 점자, 텍스트 영역을 인식한 뒤 음성 안내를 제공하는 것을 목표로 한다. 따라서 실제 사용 환경을 반영할 수 있도록 손가락이 포함된 이미지와 포함되지 않은 이미지를 모두 수집하며, 조명, 거리, 각도 변화에 강인한 모델 학습을 목표로 한다.

- 데이터셋 폴더: [Google Drive](https://drive.google.com/drive/folders/14unlFZW-sjBtWq7clYM9Dg_nzaNkbi2b?usp=drive_link)
- Roboflow 프로젝트 유형: `Object Detection`
- 라벨링 방식: 객체별 Bounding Box

---

## 수집 대상

수집 대상은 동화책 내 그림 객체, 촉각 교구, 점자, 텍스트 영역이다.

| 구분 | 수집 대상 |
| --- | --- |
| 책 속 그림 객체 | 원숭이, 꽃, 화분, 돌 |
| 촉각 교구 | 원숭이 교구, 꽃 교구, 화분 교구, 돌 교구 |
| 문해 보조 요소 | 점자 영역, 일반 텍스트 영역 |
| 실제 사용 상황 | 손가락으로 객체를 가리키는 장면 |

동일한 객체라도 배경, 촬영 조건, 객체 위치, 가림 정도가 다양한 이미지를 포함한다. 객체가 화면 일부만 보이는 경우도 실제 사용 환경에서 발생할 수 있으므로 수집 대상에 포함한다.

---

## 클래스 목록

객체 탐지 데이터셋에는 다음 10개 클래스를 사용한다.

| 클래스 | 정의 |
| --- | --- |
| `book_monkey` | 책에 인쇄된 원숭이 그림 |
| `book_flower` | 책에 인쇄된 꽃 그림 |
| `book_flowerpot` | 책에 인쇄된 화분 그림 |
| `book_stone` | 책에 인쇄된 돌 그림 |
| `tactile_monkey` | 촉각 교구 형태의 원숭이 객체 |
| `tactile_flower` | 촉각 교구 형태의 꽃 객체 |
| `tactile_flowerpot` | 촉각 교구 형태의 화분 객체 |
| `tactile_stone` | 촉각 교구 형태의 돌 객체 |
| `braille` | 점자 영역 전체 |
| `text` | 일반 인쇄 텍스트 영역 |

손가락은 입력 상황을 구성하는 요소로만 사용하고, 탐지 클래스에는 포함하지 않는다.

## 촬영 계획

### 동화책 및 촉각 교구 선정

데이터 수집 전에 사용할 동화책 페이지와 촉각 교구를 확정한다.

- 객체 클래스 10개가 모두 포함되는 페이지 또는 장면을 우선 선정한다.
- 책 속 그림 객체와 촉각 교구가 명확히 구분되도록 배치한다.
- 실제 서비스 환경처럼 책상 위 또는 웹캠 상단 촬영 구도를 기준으로 촬영한다.
- 촬영 대상 외 개인정보, 얼굴, 불필요한 주변 물체가 포함되지 않도록 정리한다.

### 클래스별 촬영 기준

각 클래스는 최소 100장 이상의 데이터를 확보하는 것을 권장한다. 클래스 간 데이터 수량이 크게 차이나지 않도록 관리한다.

| 클래스 그룹 | 권장 수집 기준 |
| --- | --- |
| 책 속 그림 객체 | 객체별 100장 이상 |
| 촉각 교구 | 객체별 100장 이상 |
| 점자 영역 | 100장 이상 |
| 텍스트 영역 | 100장 이상 |

한 이미지에 여러 객체가 함께 포함될 수 있으며, 이 경우 이미지 안의 모든 대상 객체를 각각 라벨링한다.

---

## 촬영 조건

### 조명 환경

다양한 조명 조건을 포함하여 모델이 밝기 변화에 강인하도록 구성한다.

- 자연광
- 실내 조명
- 저조도 환경
- 그림자 또는 약한 역광이 포함된 환경

단, 객체나 페이지를 식별할 수 없을 정도로 과도하게 어둡거나 흔들린 이미지는 학습 데이터로 사용하지 않는다.

### 거리 및 각도

실제 사용자의 촬영 습관과 웹캠 설치 환경을 반영하여 다양한 거리와 각도로 촬영한다.

- 정면
- 사선
- 탑뷰
- 가까운 거리
- 중간 거리
- 페이지 전체가 보이는 거리

객체가 화면 중앙에 있는 이미지뿐 아니라 가장자리, 일부 잘림, 부분 가림이 있는 이미지도 포함한다.

### 실제 사용 환경 데이터

다음과 같은 실제 입력 상황을 일부 포함한다.

- 약한 흐림
- 약한 기울어짐
- 손이나 촉각 교구에 의한 부분 가림
- 책 페이지 접힘 또는 곡면
- 웹캠 상단 촬영과 유사한 시점
- 책과 촉각 교구가 함께 놓인 상태

촬영 실패 수준의 심한 흐림, 과노출, 과도한 가림은 별도로 분리하고 라벨링 대상에서 제외한다.

---

## 손가락 포함 데이터

손가락 포함 데이터는 사용자가 특정 객체를 가리키는 실제 서비스 상황을 반영하기 위해 수집한다.

수집 상황은 다음을 포함한다.

- 그림 가리키기
- 촉각 교구 가리키기
- 점자 가리키기
- 텍스트 가리키기

손가락이 객체 일부를 가리는 이미지를 포함하되, Bounding Box는 실제로 보이는 객체 영역을 기준으로 지정한다. 손끝, 손가락, 손바닥은 별도 객체로 라벨링하지 않는다.

---

## 손가락 미포함 데이터

손가락 미포함 데이터는 객체 자체의 기준 형태를 학습하기 위해 수집한다.

- 객체가 선명하게 보이는 기준 이미지
- 다양한 배경 위에 놓인 촉각 교구 이미지
- 책 페이지 전체 또는 일부가 보이는 이미지
- 점자와 텍스트 영역이 명확하게 보이는 이미지

손가락 포함 데이터와 손가락 미포함 데이터가 한쪽으로 치우치지 않도록 균형 있게 관리한다.

---

## Roboflow 업로드 및 정리

수집한 이미지는 Google Drive에 1차 정리한 뒤 Roboflow에 업로드하여 라벨링과 증강을 진행한다.

### 업로드 전 정리

- 중복 이미지 제거
- 심하게 흔들리거나 객체 식별이 어려운 이미지 제외
- 클래스별 이미지 수량 확인
- 손가락 포함/미포함 이미지 비율 확인
- 조명, 거리, 각도 조건이 편중되지 않았는지 확인
- 개인정보 또는 얼굴이 포함된 이미지 제거

### Roboflow 작업

- 프로젝트 유형은 `Object Detection`으로 설정한다.
- 클래스명은 본 문서의 클래스 목록과 동일하게 입력한다.
- 이미지 내 모든 대상 객체에 Bounding Box를 지정한다.
- 책 속 그림 객체(`book_`)와 촉각 교구(`tactile_`)를 구분하여 라벨링한다.
- 점자는 점자 영역 전체를 `braille`로 라벨링한다.
- 텍스트는 문장 또는 문단 단위로 `text`로 라벨링한다.
- 라벨링 완료 후 train, validation, test 데이터로 분할한다.
- 필요 시 Roboflow 증강 기능을 사용하여 밝기, 회전, 기울어짐 등 실제 환경 변화를 보강한다.

## 데이터 품질 관리 기준

수집 후 다음 기준으로 데이터셋 품질을 점검한다.

| 점검 항목 | 기준 |
| --- | --- |
| 클래스 수량 | 각 클래스 최소 100장 이상 권장 |
| 클래스 균형 | 특정 클래스에 데이터가 과도하게 몰리지 않도록 관리 |
| 촬영 조건 | 조명, 거리, 각도 조건이 다양하게 포함되어야 함 |
| 손가락 포함 여부 | 손가락 포함/미포함 데이터가 모두 존재해야 함 |
| 객체 다양성 | 동일 객체라도 배경, 위치, 가림 정도가 다양해야 함 |
| 라벨 정확도 | 객체 전체 또는 보이는 영역에 맞게 Bounding Box 지정 |
| 개인정보 | 얼굴, 이름, 개인 식별 정보가 포함되지 않아야 함 |
