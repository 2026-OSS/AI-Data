# YOLO11n v6 Class Performance Analysis

## 1. 작업 목적

본 문서는 Roboflow v6 데이터셋으로 학습한 YOLO11n v6 모델의 클래스별 성능을 분석하고, 성능이 낮은 클래스를 중심으로 데이터 보완 및 후속 개선 방향을 정리하기 위해 작성하였다.

본 이슈의 비교 범위는 다른 데이터셋 버전 또는 다른 모델과의 비교가 아니라, **YOLO11n v6 모델 내부의 클래스별 성능 비교**로 한정한다.

## 3. 전체 성능 요약

| Precision | Recall | mAP@0.5 | mAP@0.5:0.95 |
|---:|---:|---:|---:|
| 0.835 | 0.719 | 0.776 | 0.545 |

YOLO11n v6 모델은 전체 기준 Precision 0.835, Recall 0.719, mAP@0.5 0.776, mAP@0.5:0.95 0.545의 성능을 보였다.

Precision은 비교적 높은 편이지만 Recall이 0.719로 낮아, 실제 서비스 환경에서는 사용자가 손가락으로 가리킨 객체가 탐지되지 않을 가능성이 있다. 또한 mAP@0.5에 비해 mAP@0.5:0.95가 낮으므로 일부 클래스에서는 객체 위치는 찾더라도 bbox 경계 정밀도가 부족할 수 있다.

| `text` | 텍스트 영역은 mAP@0.5:0.95가 낮아 bbox 경계 정밀도 보완이 필요함 | 텍스트 박스 기준을 통일하고, 글자 영역 크기/배경이 다른 이미지 추가 |

## 7. 주요 해석

### 7.1 `book_flower`

`book_flower`는 mAP@0.5가 0.995로 매우 높아 객체 존재 여부는 안정적으로 탐지된다. 다만 mAP@0.5:0.95는 0.539로 상대적으로 낮으므로, bbox 경계를 더 정밀하게 맞추는 방향의 라벨 검수가 필요하다.

### 7.2 `braille`

`braille`은 mAP@0.5 0.913, mAP@0.5:0.95 0.758로 전체 클래스 중 가장 안정적인 성능을 보인다. 현재 데이터 기준으로는 우선 개선 대상에서 제외해도 된다.

### 7.3 `book_stone`

`book_stone`은 mAP@0.5:0.95가 0.342로 가장 낮다. 이는 객체를 어느 정도 찾더라도 bbox 위치가 정확하지 않을 가능성이 크다는 의미다. 돌 객체의 크기, 각도, 배경 변화가 있는 데이터를 추가하고 bbox 경계를 다시 검수해야 한다.

### 7.4 `tactile_flower`

`tactile_flower`는 mAP@0.5가 0.495로 가장 낮아 탐지 자체가 불안정하다. 촉각 꽃 객체가 단독으로 있는 이미지뿐 아니라 다른 촉각 객체와 가까이 배치된 이미지도 추가해야 한다.

### 7.5 `text`

`text`는 mAP@0.5:0.95가 0.450으로 낮다. 텍스트 영역은 라벨 경계 기준이 흔들리기 쉬우므로, 텍스트 박스를 어느 범위까지 포함할지 기준을 통일하는 것이 필요하다.

## 8. 개선 우선순위

| 우선순위 | 클래스 | 이유 |
|---|---|---|
| 1순위 | `book_stone` | mAP@0.5:0.95가 가장 낮아 bbox 정밀도 문제가 큼 |
| 2순위 | `tactile_flower` | mAP@0.5가 가장 낮아 탐지 자체가 불안정함 |
| 3순위 | `text` | bbox 경계 기준이 불안정하여 위치 정밀도 보완 필요 |
| 4순위 | `book_monkey`, `book_flowerpot` | 기본 탐지는 가능하지만 bbox 정밀도 개선 필요 |

## 9. 데이터 보완 방향

- `book_stone`, `tactile_flower` 클래스의 이미지 수를 우선적으로 보강한다.
- 가까운 객체가 함께 배치된 이미지를 추가하여, 실제 서비스 환경에서 여러 객체가 동시에 인식되는 상황을 반영한다.
- bbox 경계를 더 타이트하게 재검수한다.
- 텍스트 영역은 라벨링 기준을 통일한다.
- 손끝으로 객체를 가리키는 서비스 구조를 고려하여, 객체 일부가 손에 가려지는 경우도 데이터에 포함한다.

## 10. 결론

YOLO11n v6 모델은 전체적으로 기본 객체 탐지는 가능하지만, 클래스별 성능 편차가 존재한다. 특히 `book_stone`, `tactile_flower`, `text`는 실제 서비스 적용 전 개선이 필요한 클래스다.

따라서 #37 작업은 다른 모델 버전과의 비교가 아니라, YOLO11n v6 내부의 클래스별 성능 분석 및 개선 대상 선정으로 정리할 수 있다. 최종 개선 대상은 `book_stone`, `tactile_flower`, `text`로 선정한다.
